## 二、各文件功能设计目标

### 2.1 `main.c`

**职责：** 程序入口、初始化流程、主事件循环、退出清理

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 启动初始化 | 调用各模块 `*_init()`，加载配置，检测终端尺寸 |
| 主循环 | 等待输入事件 → 分发到 editor/menu → 调用 display_flush 刷新 |
| 退出流程 | 检查未保存提示 → 保存配置 → 调用各模块 `*_exit()` |
| 命令行参数 | 支持 `edit filename` 启动时打开指定文件 |

**依赖：** editor, display, input, config, platform

---

### 2.2 `document.c` / `document.h`

**职责：** 文档数据模型、文本存储、行编辑、撤销/重做

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 行存储结构 | `struct Line { char *text; int len; int cap; }` 或 `char **lines` 动态数组 |
| 插入字符 | 在指定 (row, col) 插入字符，memmove + realloc |
| 删除字符 | Backspace/Delete 删除前后字符 |
| 换行 | Enter 时拆分当前行，插入新行 |
| 合并行 | Backspace 在行首时与上一行合并 |
| 获取行/列 | `document_get_line()`, `document_get_char()`, `document_line_count()` |
| 撤销栈 | 操作记录 `{type, row, col, old_str, new_str}`，undo/redo |
| 修改标志 | `document_modified` 用于保存前提示 |
| 最大行/列限制 | 可配置的文档大小上限 |

**扩展功能（Phase 5/6）：**
- 编码信息：`encoding`（UTF-8/GBK）、`has_utf8_bom`，加载时检测，保存时还原
- UTF-8 码点删除：删除/退格时按码点而非字节删除

**接口示例：**
```c
Document* document_new(void);
void document_free(Document *doc);
int document_load(Document *doc, const char *path);
int document_save(Document *doc, const char *path);
void document_insert_char(Document *doc, int row, int col, char c);
void document_delete_char(Document *doc, int row, int col, int backward);
void document_break_line(Document *doc, int row, int col);
void document_merge_line(Document *doc, int row);
int document_undo(Document *doc);
int document_redo(Document *doc);
```

---

### 2.3 `display.c` / `display.h`

**职责：** 显示抽象层、屏幕缓冲、双缓冲、平台接口封装

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 平台无关接口 | display_init, display_exit, display_get_size, display_put_cell 等（见功能需求 5.7） |
| 屏幕缓冲区 | Cell 二维数组（字符+属性），当前帧 + 后备帧 |
| 双缓冲 | 写入后备帧，display_flush 时 diff 输出变更单元 |
| 脏区域优化 | dirty_rect 记录，仅重绘变更区域 |
| 字符属性 | ATTR_NORMAL, ATTR_SELECT, ATTR_LINENO 等常量 |
| 光标 | display_set_cursor, display_show_cursor |

**不包含：** 布局计算、坐标换算、编辑区具体绘制逻辑（由 viewport 负责）

---

### 2.4 `viewport.c` / `viewport.h`

**职责：** 视口管理、自动换行/横滚模式、逻辑↔屏幕坐标换算、编辑区渲染

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 视口状态 | view_top_row, view_top_display_row, view_left_col |
| 显示模式 | WrapMode (WRAP_WORD / WRAP_NONE)，word_wrap 可配置 |
| 逻辑→屏幕 | logic_to_screen(row, col, &y, &x) |
| 屏幕→逻辑 | screen_to_logic(y, x, &row, &col) |
| 折行缓存 | display_line_offset[]，逻辑行到累计显示行偏移 |
| 编辑区列宽 | edit_cols = 屏宽 - 行号宽 - 滚动条宽 |
| 渲染编辑区 | 按当前模式调用 document 取行，逐字符 display_put_cell，处理选择高亮 |
| 行号列 | 首显示行输出行号，续行空或重复 |
| 滚动条 | 垂直条、横滚模式下的水平条 |
| 确保光标可见 | 光标移动后调整 view_* 使光标在视口内 |
| 模式切换 | viewport_toggle_wrap()，重置视口、失效缓存 |
| UTF-8 感知渲染 | 按码点宽度渲染（ASCII=1，CJK=2） |
| Tab 展开 | `\t` 按 tab_size 展开为空格显示 |
| 语法高亮集成 | 接收 SyntaxContext，合并选区/语法/默认属性 |

**依赖：** document, display, config, syntax

---

### 2.5 `editor.c` / `editor.h`

**职责：** 编辑逻辑、光标、选择、剪贴板操作、命令分发

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 光标位置 | cursor_row, cursor_col，边界 clamp |
| 选择范围 | selection_start, selection_end (row,col)，规范化 start<=end |
| 键盘导航 | 方向键（含行首/尾 DOS 风格跳转）、Home/End、Ctrl+Home/End、Page Up/Down、Ctrl+←/→（WRAP_NONE 模式下水平滚动视口） |
| 选择扩展 | Shift+方向键扩展 selection_end |
| 插入/覆盖模式 | insert_mode，Insert 键切换 |
| 删除选中 | 有选择时 Delete/Backspace 删除选区 |
| 剪切/复制/粘贴 | 调用 clipboard，并调用 document 的删除/插入 |
| 全选 | selection 设为 (0,0) 到 (last_row, last_col) |
| 与 document 交互 | 所有编辑操作通过 document 接口 |
| 与 viewport 交互 | 移动光标后调用 viewport_ensure_cursor_visible |
| 语法高亮上下文 | SyntaxContext，管理当前文件的语法定义与块注释状态（Phase 6） |
| 自动语法匹配 | 打开文件时按扩展名自动加载对应 INI 语法配置 |
| 语法增量重建 | 每次编辑后从光标行起增量重建块注释状态（Phase 7） |

**依赖：** document, viewport, clipboard, syntax

---

### 2.6 `input.c` / `input.h`

**职责：** 键盘与鼠标输入、事件归一化

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 键盘事件 | 读取扫描码/虚拟键，支持方向键、功能键、Ctrl/Alt/Shift 组合 |
| 鼠标事件 | 左/右键按下/释放、移动，获取 (x, y) |
| 事件结构 | `typedef struct { int type; int key; int mod; int mx, my; } InputEvent` |
| 非阻塞读取 | input_poll() 或 input_wait_event()，平台相关 |
| DOS 鼠标 | INT 33h 初始化、状态读取 |
| Windows 控制台 | ReadConsoleInput |
| Linux | POSIX termios 原始模式、`read()` 阻塞读取；鼠标通过 ioctl/ANSI 协议（无 ncurses） |

**接口示例：**
```c
void input_init(void);
void input_exit(void);
int input_poll(InputEvent *ev);
void input_translate_screen_to_logic(int sx, int sy, int *row, int *col);  // 调用 viewport
```

### 2.7 `menu.c` / `menu.h`

**职责：** 菜单栏、下拉菜单、对话框、菜单项与命令绑定

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 菜单栏 | File / Edit / Search / View / Help 水平项，Alt+字母激活 |
| 下拉菜单 | 展开后显示子项，方向键选择，Enter 执行 |
| 菜单绘制 | 高亮当前项（反色），display_put_cell 逐字符 |
| 菜单点击 | 判断 (mx, my) 是否在菜单项矩形内，执行对应命令 |
| 对话框 | 查找、替换、打开、另存为、确认退出等弹窗 |
| 输入框 | 对话框内文本输入，支持 Backspace、Enter 确认、Esc 取消 |
| 错误提示 | 弹窗显示 "无法打开文件" 等消息 |
| 保存前确认 | "是否保存更改？ Yes/No/Cancel" |
| 状态栏 | 显示文件名、行/列、模式、换行、编码、语言 |
| View > Set Language... | 弹出列表选择对话框切换语法高亮（Phase 7） |

**命令到实现的映射：** 菜单/快捷键触发 → 调用 editor、document、search 等接口

### 2.8 `search.c` / `search.h`

**职责：** 查找、替换、查找选项

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 查找 | 从 (row, col) 起在 document 中查找 str，返回匹配位置 |
| 查找下一个 | 从当前匹配后继续查找， wrap 到文档开头 |
| 查找上一个 | 反向查找 |
| 替换 | 替换当前匹配或全部匹配 |
| 查找选项 | case_sensitive, whole_word（可选） |
| 高亮匹配 | 可选：在编辑区临时高亮当前匹配 |

**接口示例：**
```c
void search_init(const char *find_str, const char *replace_str, int options);
/* from_row/from_col: 搜索起点（输入）；out_row/out_col: 匹配位置（输出）；返回匹配长度，-1=未找到 */
int search_next(Document *doc, int from_row, int from_col, int *out_row, int *out_col);
int search_prev(Document *doc, int from_row, int from_col, int *out_row, int *out_col);
/* match_len: 当前匹配的字符长度，用于确定需要删除多少字符再插入替换串 */
int search_replace_current(Document *doc, int row, int col, int match_len);
int search_replace_all(Document *doc);
```

### 2.9 `clipboard.c` / `clipboard.h`

**职责：** 内部剪贴板（不依赖系统剪贴板，便于 DOS 兼容）

| 实现目标 | 对应功能需求 |
|----------|--------------|
| 存储 | `char *clipboard` 动态数组，可含 `\n` |
| 复制 | clipboard_set(text, len) |
| 粘贴 | clipboard_get() 返回内容，调用方负责插入 document |
| 清空 | 退出或复制新内容时 |
| 多行支持 | 粘贴时按 `\n` 拆行插入 |

**接口示例：**
```c
void clipboard_set(const char *text, int len);
const char* clipboard_get(int *out_len);
int clipboard_is_empty(void);
```

### 2.10 `config.c` / `config.h`

**职责：** 配置存储、默认值、启动加载与退出保存

| 实现目标 | 对应功能需求 |
|----------|--------------|

**接口示例：**

### 2.17 `platform/` 平台实现

**职责：** 将 display、input 等抽象接口映射到具体平台 API

#### `platform.h`

定义平台抽象接口（display_*, input_* 的声明），以及 `#ifdef` 选择当前平台。

#### `platform_win.c`（Windows 控制台）

| 实现目标 | 对应功能需求 |
|----------|--------------|
| display | SetConsoleCursorPosition, WriteConsoleOutput, SetConsoleTextAttribute |
| 键盘 | ReadConsoleInput 读取 KEY_EVENT |
| 鼠标 | ReadConsoleInput 读取 MOUSE_EVENT，启用 ENABLE_MOUSE_INPUT |
| 尺寸 | GetConsoleScreenBufferInfo |

#### `platform_dos.c`（FreeDOS / MS-DOS）

| 实现目标 | 对应功能需求 |
|----------|--------------|
| display | INT 10h 设置模式、写字符、设置光标；直接写 B800:0 显存 |
| 键盘 | INT 16h 读取扫描码 |
| 鼠标 | INT 33h 初始化、获取位置与按键状态 |
| 文件 | DOS INT 21h 或标准 C 库（若存在） |

#### `platform_nix.c`（Linux / Unix 终端）

| 实现目标 | 对应功能需求 |
|----------|--------------|
| display | termios 原始模式，`write()` 输出 ANSI 转义（光标、颜色），无 ncurses |
| 键盘 | `read()` 从 stdin 读取键码 |
| 鼠标 | 发送 `\033[?1000h` 启用 ANSI X10 鼠标协议（依赖终端支持）；不支持时自动降级为纯键盘操作 |
| 尺寸 | ioctl TIOCGWINSZ 或环境变量 |

### 5.3 测试框架与断言

采用**极简自实现**，**零第三方依赖**（不使用 Unity、Check 等），仅依赖 C 标准库（stdio、stdlib、string），便于所有目标平台（含 DOS）编译：

```c
// test_runner.c 核心
/* 断言失败时记录错误并立即返回（同一函数内后续断言不再执行）*/
#define ASSERT_EQ(a, b) do { \
    if ((a) != (b)) { \
        fprintf(stderr, "FAIL %s:%d: %d != %d\n", __FILE__, __LINE__, (int)(a), (int)(b)); \
        fail_count++; return; \
    } else { pass_count++; } \
} while(0)

#define ASSERT_STR_EQ(a, b) do { \
    if (strcmp((a), (b)) != 0) { \
        fprintf(stderr, "FAIL %s:%d: \"%s\" != \"%s\"\n", __FILE__, __LINE__, (a), (b)); \
        fail_count++; return; \
    } else { pass_count++; } \
} while(0)

static int pass_count, fail_count;

void test_document_run(void);
void test_clipboard_run(void);
void test_search_run(void);
void test_viewport_run(void);
void test_util_run(void);
void test_config_run(void);
void test_editor_run(void);
void test_utf8_run(void);
void test_encoding_run(void);
void test_syntax_run(void);

int main(void) {
    test_document_run();
    test_clipboard_run();
    test_search_run();
    test_viewport_run();
    test_util_run();
    test_config_run();
    test_editor_run();
    test_utf8_run();
    test_encoding_run();
    test_syntax_run();
    printf("%d passed, %d failed\n", pass_count, fail_count);
    return fail_count > 0 ? 1 : 0;
}
```

```makefile
TEST_SRCS = tests/test_runner.c tests/test_document.c tests/test_clipboard.c \
```
