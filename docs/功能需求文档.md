# DOS 文本编辑器 - 功能需求文档

中文 | [English](FUNCTIONAL_REQUIREMENTS.md)

> 类似 MS-DOS Edit 的字符界面文本编辑器功能规格说明

---

## 一、项目概述

开发一个可在 DOS/终端环境下运行的纯文本编辑器，采用字符界面（非图形界面），支持键盘与鼠标两种操作方式完成编辑功能。目标用户为需要在命令行环境中快速编辑配置文件和脚本的用户。

## 三、非功能需求

### 3.1 运行环境

- **目标平台**：DOS（如 FreeDOS）、Windows 命令提示符、Linux/Unix 终端
- **实现语言**：仅限 **C 或 C++**，不使用其他语言
- **依赖**：**零第三方依赖**，仅使用：
  - **C 标准库**：stdio、stdlib、string、ctype、stdbool 等
  - **平台原生 API**：DOS INT 10h/16h/21h/33h、Windows Console API、POSIX termios/ioctl
  - 不链接 ncurses、Qt、SDL、Check、Unity 等任何第三方库；测试亦仅用自实现断言
- **Ctrl+C 信号处理**：在 Windows 下须通过 `SetConsoleCtrlHandler` 屏蔽默认 SIGINT，使 Ctrl+C 可被程序作为"复制"使用；在 Linux/DOS 下须在 termios 初始化时设置 `c_lflag &= ~ISIG`，屏蔽信号传递，否则 Ctrl+C 将直接终止进程而非触发复制功能

### 3.2 性能要求

- 启动时间：< 2 秒
- 打开 100KB 文件：< 1 秒
- 键盘响应：无明显延迟

### 3.3 兼容性

- 支持 80×25、80×50 等常见终端尺寸
- 在窗口大小变化时能正确处理（如终端调整大小时）

### 3.4 可靠性

- 异常退出时尽量不损坏原文件
- 保存时建议先写临时文件，成功后再覆盖原文件

## 五、显示系统设计（核心架构）

> 显示系统是字符界面编辑器的底层基础，所有视觉输出均通过此系统完成。采用分层抽象，便于跨平台移植。

### 5.1 分层架构

```
┌─────────────────────────────────────────────────────────┐
│  应用层：编辑器逻辑（文档、光标、选择、菜单状态）            │
├─────────────────────────────────────────────────────────┤
│  渲染层：将逻辑状态转换为屏幕内容（布局、坐标换算、重绘）     │
├─────────────────────────────────────────────────────────┤
│  输出抽象层：统一接口（putchar、set_cursor、set_attr 等）   │
├─────────────────────────────────────────────────────────┤
│  平台实现层：DOS INT 10h / Windows Console / POSIX termios+ANSI  │
└─────────────────────────────────────────────────────────┘
```

### 5.2 屏幕缓冲区与双缓冲


| 设计项       | 实现方式                                                               |
| --------- | ------------------------------------------------------------------ |
| **显存/缓冲** | 维护全屏字符缓冲区 `char screen[rows][cols*2]`（字符+属性），或等价结构                 |
| **双缓冲**   | 每次重绘先写入「后备缓冲区」，完成后一次性与当前屏幕 diff，只输出变更的字符单元，减少闪烁和 I/O               |
| **脏区域**   | 记录 `dirty_rect`（如 top_row, left_col, bottom_row, right_col），仅重绘脏区域 |
| **全屏重绘**  | 启动时、窗口尺寸变化时、切换模式时执行                                                |


**关键数据结构示例（C）：**

```c
typedef struct {
    char    ch;      /* 单字节字符（第一阶段仅支持 ASCII/ANSI，不支持多字节编码） */
    uint8_t attr;    /* 属性：前景色(4bit)+背景色(4bit) */
} Cell;
/*
 * 注意：若未来扩展支持 UTF-8 或 GBK，需修改此结构（如改为 uint16_t ch 或
 * wchar_t），并同步修改所有渲染逻辑。这是一次架构性变更，不是简单扩展。
 */

typedef struct {
    int rows, cols;
    Cell *buffer;    // 当前帧
    Cell *back;      // 后备帧，用于 diff
    bool *dirty;     // 可选：每格脏标记
} ScreenBuffer;
```

### 5.3 坐标与视口


| 概念                | 定义                     | 实现要点                                                                                                                |
| ----------------- | ---------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **逻辑坐标**          | 文档中的 (row, col)，从 0 开始 | 行可超长，col 为逻辑列（含行号占位）                                                                                                |
| **屏幕坐标**          | 物理行列 (y, x)，对应终端字符格    | 编辑区左上角为 (menu_h, line_num_w)，若显示行号                                                                                  |
| **视口 (Viewport)** | 当前可见的文档行范围             | `view_top_row`：视口首行在文档中的行号；可见行数 = `rows - menu_h - status_h`（如 80×25 下 = 25-1-1 = **23 行**编辑区，不再多减 1；若显示水平滚动条则再减 1） |
| **换算**            | 逻辑→屏幕                  | 受「自动换行」模式影响，详见 5.9 节                                                                                                |


**垂直滚动：** 光标超出视口时更新 `view_top_row`，使光标位于可视区中部或边缘（可配置）。

### 5.4 编辑区渲染流程

```
1. 根据当前模式（自动换行/横滚）确定视口与绘制逻辑（见 5.9）
2. 根据 view_top_row（及 view_left_col，横滚模式）确定要绘制的行列范围
3. 对每行：
   a. 若有行号：先渲染行号列（右对齐，固定宽度）
   b. 渲染文本：从左到右逐字符，处理选择高亮（选中字符用反色 attr）
   c. 超长行：按 5.9 节规则处理（折行或水平裁剪）
4. 若行数不足填满编辑区：剩余行用空格+默认属性填充
5. 光标：将逻辑 (row,col) 换算为屏幕 (y,x) 后写入光标
6. 若有滚动条：垂直条 + 横滚模式下的水平条
```

### 5.7 平台输出接口抽象

统一抽象层接口（便于 DOS / Windows / Linux 共用逻辑）：

```c
// display.h - 平台无关接口
void display_init(void);           // 初始化，获取 rows/cols
void display_exit(void);
void display_get_size(int *rows, int *cols);
void display_put_cell(int y, int x, char c, uint8_t attr);
void display_set_cursor(int y, int x);
void display_show_cursor(bool show);
void display_flush(void);          // 将后备缓冲提交到屏幕
void display_clear(uint8_t attr);  // 全屏清空为某属性
```

**平台实现：**

- **DOS**：INT 10h（模式设置、写字符、设置光标）、直接写 B800:0 显存
- **Windows CMD**：`SetConsoleCursorPosition`、`WriteConsoleOutput`、`SetConsoleTextAttribute`
- **Linux**：POSIX termios 设置原始模式，`write()` 输出 ANSI 转义序列控制光标与属性，无第三方库

### 5.8 鼠标光标与点击映射


| 项目          | 实现方式                                                |
| ----------- | --------------------------------------------------- |
| **鼠标显示**    | 软件光标：在鼠标所在 (y,x) 绘制自定义字符（如 `▶`）或块；或隐藏系统光标，用高亮块表示    |
| **点击→逻辑坐标** | 需结合当前显示模式（5.9），将屏幕 (sx,sy) 反算为逻辑 (row,col)          |
| **边界检查**    | 若 `doc_row` 超出文档行数，则 clamp 到最后一行；`doc_col` 不可超过该行长度 |


### 5.9 自动换行与水平滚动（双模式设计）

> 编辑器必须同时支持**自动换行**和**取消自动换行（屏幕横滚）**两种模式，用户可通过菜单或快捷键切换。

#### 5.9.1 模式定义


| 模式                   | 行为                                           | 适用场景       |
| -------------------- | -------------------------------------------- | ---------- |
| **字符折行 (Char Wrap)** | 长行按编辑区列宽强制折成多「显示行」（按字符位置截断，不保证在词边界断开），无需横向滚动 | 阅读长段落、窄屏   |
| **横滚 (No Wrap)**     | 每逻辑行仅占一屏幕行，超长部分通过水平滚动查看                      | 编辑宽表、代码、长行 |


**状态变量：** `enum { WRAP_CHAR, WRAP_NONE }`（`WRAP_CHAR` 即字符折行，`WRAP_NONE` 即横滚模式），存入配置，启动时恢复。

#### 5.9.2 可显示列宽

编辑区可用于显示文本的列数（扣除行号、滚动条）：

```
edit_cols = screen_cols - line_num_width - (vertical_scrollbar ? 1 : 0)
```

自动换行时，每行的「折行宽度」= `edit_cols`。横滚模式时，单行逻辑长度可远大于 `edit_cols`。

#### 5.9.3 自动换行模式


| 项目            | 实现方式                                                                                                                                                                                                    |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **逻辑行→显示行**   | 一行逻辑长度为 `len` 时，完整公式为：`display_lines = (len == 0) ? 1 : (len + edit_cols - 1) / edit_cols`；空行必须特判为 1，因向上取整公式对 len=0 结果为 0                                                                               |
| **逻辑坐标→屏幕坐标** | `(row, col)` → 先算该行前占用的显示行数 `offs = sum(display_lines(0..row-1))`，再算本行内偏移 `sub = col / edit_cols`，`screen_row = view_top_row_offset + offs + sub - view_top_display_row`；`screen_col = col % edit_cols` |
| **屏幕坐标→逻辑坐标** | 反向：由 `screen_row` 累加各逻辑行 `display_lines` 得到 `row`，`col = screen_col + (sub_offset * edit_cols)`                                                                                                         |
| **光标/选择**     | 光标可落在任意逻辑 (row,col)，换算到屏幕 (y,x) 显示；选择范围在逻辑坐标下计算，渲染时对落在选区内的字符反色                                                                                                                                          |
| **垂直滚动**      | `view_top_row` 表示「视口顶部对应的文档逻辑行号」；在折行模式下，需引入 `view_top_sub`（逻辑行内折行偏移，0 表示该行首屏）或等价：`view_top_display_row`（从 0 计数的「显示行」偏移）                                                                                 |
| **行号**        | 同一逻辑行被折成多显示行时，仅在首显示行显示行号，续行行号列为空或重复（由产品决定）                                                                                                                                                              |


**建议存储：** 预计算或缓存 `display_line_offset[row]` = 第 row 行之前累计的显示行数，便于 O(1) 逻辑↔屏幕换算。

#### 5.9.4 横滚模式（取消自动换行）


| 项目          | 实现方式                                                                                                                                    |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **逻辑行↔显示行** | 1 逻辑行 = 1 显示行，一一对应                                                                                                                      |
| **水平视口**    | 新增 `view_left_col`：视口左边缘对应的逻辑列号；可见列数 = `edit_cols`                                                                                      |
| **逻辑→屏幕**   | `screen_row = doc_row - view_top_row`；`screen_col = doc_col - view_left_col`（若 doc_col 在 [view_left_col, view_left_col+edit_cols) 内则可见） |
| **屏幕→逻辑**   | `doc_row = view_top_row + screen_row`；`doc_col = view_left_col + screen_col`                                                            |
| **水平滚动**    | 光标右移超出视口右边缘时，`view_left_col += 1` 或按步长增加；左移同理。Page Left/Right 可一次移动 `edit_cols`                                                         |
| **滚动条**     | 水平滚动条长度表示「当前可见列宽 / 该行总长」；若多行长度不一，可按「最长行」或「当前行」计算，产品可定                                                                                   |
| **超长行渲染**   | 只绘制 `line[view_left_col .. view_left_col+edit_cols-1]` 的区间，左侧被裁掉                                                                        |


#### 5.9.5 模式切换时的处理


| 项目       | 实现方式                                                                                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **切换瞬间** | 从 WRAP_CHAR→WRAP_NONE：清空 `view_top_sub`，设 `view_left_col = 0`（之前处于折行模式，水平偏移无意义，归零）；从 WRAP_NONE→WRAP_CHAR：设 `view_left_col = 0`，按新规则重算 `view_top_display_row` |
| **光标保持** | 逻辑 (row,col) 不变，仅重新换算屏幕位置；若横滚下 `view_left_col` 过大导致光标不可见，则调整 `view_left_col` 使光标进入视口                                                                         |
| **重绘**   | 切换模式后触发全编辑区重绘                                                                                                                                                |


#### 5.9.6 数据结构与接口

```c
// 显示模式
typedef enum { WRAP_CHAR, WRAP_NONE } WrapMode; /* WRAP_CHAR=字符折行，WRAP_NONE=横滚 */

// 视口状态（横滚模式）
int view_left_col;        // 水平偏移，仅 WRAP_NONE 有效

// 视口状态（折行模式）：用「显示行」更易实现
int view_top_display_row; // 视口顶对应的「第几条显示行」(从0计)

// 折行缓存：逻辑行号 -> 该行之前累计显示行数
int *display_line_offset; // 或现场计算

// 坐标换算接口
void logic_to_screen(int row, int col, int *out_y, int *out_x);
void screen_to_logic(int y, int x, int *out_row, int *out_col);
```

### 6.2 文档模型实现


| 功能       | 实现方式                                                                                                                                                                                                                       |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **行存储**  | `char* lines[]` 或 `struct Line { char* text; int len; }` 动态数组，每行一个元素                                                                                                                                                       |
| **插入字符** | 在 `lines[row]` 的 `col` 处 `memmove` 腾位并写入，`realloc` 若需扩容                                                                                                                                                                    |
| **删除字符** | `memmove` 将 col+1 之后前移，缩小 `len`                                                                                                                                                                                            |
| **换行**   | 将当前行从 col 处拆成两行：新行 = 原行 col 之后，原行截断到 col；插入新行到 `lines[row+1]`                                                                                                                                                              |
| **合并行**  | Backspace 在行首时：将当前行接到上一行末尾，删除当前行                                                                                                                                                                                           |
| **撤销**   | 操作栈：每步存 `{type, row, col, old_text, new_text}`，撤销时反向执行；**合并策略**：连续字符输入（相邻列、无跳转）合并为一条记录；空格、Enter、删除、粘贴各自独立为一条记录；**保存点**：每次成功保存时记录 `undo_save_index = undo_top`，当 undo_top 回到该值时 `modified` 标志清为 false，避免撤销到保存状态后仍误提示「未保存更改」 |


### 6.3 第一阶段（MVP）— 实现方式


| 功能           | 实现方式                                                                |
| ------------ | ------------------------------------------------------------------- |
| **新建**       | `document_new()`：清空 `lines`，添加一个空行                                  |
| **打开**       | `fopen` 读入，按 `\n` 分行存入 `lines[]`，处理 `\r`                            |
| **保存**       | 逐行 `fwrite`，行间写入 `\n`；先写 `.tmp`，成功则 `rename` 覆盖                     |
| **退出**       | 检查 `document_modified`，若为真则弹对话框确认                                   |
| **插入/删除/换行** | 调用 document 的插入/删除/换行接口，更新光标                                        |
| **导航**       | 根据按键更新 `cursor_row`、`cursor_col`，做边界 clamp；更新 `view_top_row` 保持光标可见 |
| **菜单栏**      | 静态绘制 "File Edit Search View Help"，Alt+键高亮并打开下拉                      |
| **状态栏**      | `sprintf` 格式化 "行 %d 列 %d 共 %d 行 INS/OVR"，`display_put_cell` 逐字符输出   |
| **鼠标点击定位**   | 在输入层将点击 (x,y) 转为 (row,col)，调用 `editor_set_cursor(row,col)`          |


### 6.4 第二阶段 — 实现方式


| 功能         | 实现方式                                                            |
| ---------- | --------------------------------------------------------------- |
| **选择**     | `selection_start`、`selection_end` 两对 (row,col)，Shift+方向键时更新 end |
| **剪切**     | 复制选中内容到剪贴板（内部 `char* clipboard`），删除选中范围                         |
| **复制**     | 将选中内容拷贝到 `clipboard`                                            |
| **粘贴**     | 在光标处插入 `clipboard`，可含 `\n` 需拆行                                  |
| **鼠标拖拽选择** | 左键按下记录 start，移动时更新 end 并重绘选中高亮                                  |
| **菜单点击**   | 判断点击坐标是否落在菜单项矩形内，执行对应命令                                         |
| **查找**     | 弹窗输入查找串，从当前光标后线性扫描 `lines[]`，`strstr` 匹配                        |
| **查找下一个**  | 从 `cursor_col+1` 或下一行开始继续 `strstr`                              |
| **撤销/重做**  | 调用 document 的 undo/redo，刷新显示                                    |


### 6.5 第三阶段 — 实现方式


| 功能        | 实现方式                                                                                        |
| --------- | ------------------------------------------------------------------------------------------- |
| **替换**    | 弹窗输入查找串、替换串，定位后替换当前或全部                                                                      |
| **全部替换**  | 循环查找并替换，直到无匹配                                                                               |
| **另存为**   | 弹对话框输入新路径，按新路径执行保存逻辑                                                                        |
| **行号显示**  | 布局中为行号预留列，`sprintf("%6d", row+1)` 输出；折行模式下续行可省略或重复                                          |
| **自动换行**  | 实现 5.9.3：`display_line_offset[]` 缓存、`logic_to_screen`/`screen_to_logic`，渲染时按 `edit_cols` 折行 |
| **横滚模式**  | 实现 5.9.4：`view_left_col`、水平滚动、只渲染 `[view_left_col, view_left_col+edit_cols)` 区间             |
| **模式切换**  | `word_wrap` 状态 + 菜单/快捷键，切换时重置视口、重绘                                                          |
| **垂直滚动条** | 右列绘制，`thumb_pos = view_top_row * bar_height / total_lines`（折行时用 `view_top_display_row`）     |
| **水平滚动条** | 横滚模式下于状态栏上方或底行预留，`thumb_pos = view_left_col * bar_width / max(1, line_len)`                 |


### 6.6 第四阶段 — 实现方式


| 功能       | 实现方式                                                 |
| -------- | ---------------------------------------------------- |
| **编码支持** | 读文件时检测 BOM 或试探解码；内部用 UTF-8 或宽字符，输出时按终端能力转            |
| **语法高亮** | 渲染时按 token 类型（关键字、字符串等）设置不同 attr，可先支持 1～2 种语言        |
| **折行优化** | 大文件下 `display_line_offset` 延迟计算或按需缓存；窗口 resize 时失效重建 |
