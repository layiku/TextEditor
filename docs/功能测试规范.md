# DOS 文本编辑器 - 功能测试规范

中文 | [English](FUNCTIONAL_TEST_SPEC.md)

> 各功能的测试方法与成功/失败边界定义

---

## 一、测试说明

| 术语 | 含义 |
|------|------|
| **通过（✓）** | 满足「成功边界」全部条件，无「失败边界」情形 |
| **失败（✗）** | 出现「失败边界」任一情形，或未满足「成功边界」 |
| **手工测试** | 人工操作、肉眼观察 |
| **自动化测试** | 单元测试或脚本驱动，**零第三方依赖**，仅用 C 标准库与项目自实现断言 |
| **边界用例** | 空文档、单字符、超长行、最大文件等特殊输入 |

## 二、文件操作

### 2.1 新建文件

| 测试步骤 | 操作 |
|----------|------|
| 1 | 启动编辑器（无参数或带 `--new`） |
| 2 | 选择 File → New，或按 Ctrl+N |

| 成功边界 | 失败边界 |
|----------|----------|
| 编辑区显示空白，仅 1 行（空行） | 崩溃、黑屏、卡死 |
| 光标在 (0, 0) | 光标不可见或位置错误 |
| 状态栏显示「行 1 列 1」或等价 | 状态栏空白或数值错误 |
| 未修改标志为假，保存/退出不提示 | 错误提示「未保存」 |
| 标题/状态栏可显示「未命名」或空 | - |

---

### 2.2 打开文件

| 测试步骤 | 操作 |
|----------|------|
| 1 | 准备测试文件 `test.txt`，内容已知（如 3 行、每行 10 字符） |
| 2 | File → Open 或 Ctrl+O，输入路径 `test.txt` |
| 3 | 确认打开 |

| 成功边界 | 失败边界 |
|----------|----------|
| 编辑区显示与文件内容一致 | 内容丢失、乱码、换行错误 |
| 光标在 (0, 0) | 光标在错误位置 |
| 行数、列数正确 | 行/列统计错误 |
| 状态栏显示正确行号、列号 | - |
| 打开空文件时显示 1 空行 | 空文件导致崩溃 |
| 打开不存在的路径时弹出错误提示 | 崩溃、静默失败 |
| 打开无读权限的文件时提示错误 | 崩溃、静默失败 |
| 打开超限文件时提示或拒绝 | 静默截断、内存溢出 |

**边界用例：** 空文件、单字符、仅换行、超长单行、混合 \r\n 与 \n

---

### 2.3 保存文件

| 测试步骤 | 操作 |
|----------|------|
| 1 | 新建或打开文件，修改内容 |
| 2 | File → Save 或 Ctrl+S（已有路径）或输入新路径（另存为） |
| 3 | 用外部工具（如 `type`、`cat`）读取磁盘文件核对 |

| 成功边界 | 失败边界 |
|----------|----------|
| 磁盘文件内容与编辑区一致 | 内容不一致、截断 |
| 行尾为 `\n`（或平台规定） | `\r\n` 丢失或多余 |
| 原文件被正确覆盖（非另存为时） | 原文件损坏、残留临时文件 |
| 保存后「未保存」标志清除 | 仍提示未保存 |
| 保存到只读/无效路径时提示错误 | 崩溃、静默失败 |
| 保存过程中断（如磁盘满）不损坏原文件 | 原文件被部分覆盖或损坏 |

---

### 2.4 另存为

| 测试步骤 | 操作 |
|----------|------|
| 1 | 打开或编辑文档 |
| 2 | File → Save As，输入新路径 `other.txt` |
| 3 | 确认 |

| 成功边界 | 失败边界 |
|----------|----------|
| 新文件 `other.txt` 内容正确 | 未创建或内容错误 |
| 原文件不变（若此前有路径） | 原文件被误改 |
| 当前编辑路径更新为新路径 | 后续保存仍写到旧路径 |
| 新路径已存在时覆盖并提示（若实现） | 静默覆盖无提示 |

---

### 2.5 退出与保存前提示

| 测试步骤 | 操作 |
|----------|------|
| 1a | 修改后退出（Alt+X / Alt+F4） |
| 1b | 未修改直接退出 |
| 1c | 修改后选「不保存」退出 |
| 1d | 修改后选「取消」不退出 |

| 成功边界 | 失败边界 |
|----------|----------|
| 有修改时弹出确认（保存/不保存/取消） | 直接退出且丢失修改 |
| 选「保存」时执行保存再退出 | 选保存却未写入 |
| 选「不保存」时直接退出，不写磁盘 | 仍写入磁盘 |
| 选「取消」时留在编辑器 | 仍退出 |
| 未修改时直接退出，无弹窗 | 错误弹出确认 |

## 三、文本编辑

### 3.1 插入字符

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在空行或已有文本中间输入 `a`、`b`、`c` |
| 2 | 观察光标位置与后续字符 |

| 成功边界 | 失败边界 |
|----------|----------|
| 新字符出现在光标处，后续字符右移 | 覆盖后续字符、插入到错误位置 |
| 光标移动到新字符右侧 | 光标不动或错位 |
| 多行时只在当前行插入 | 影响其他行 |
| 行尾插入时正确扩展该行 | 换行异常、截断 |

**边界：** 行首、行尾、空文档首字符、达到最大列限时的行为（拒绝或换行）

---

### 3.2 删除字符（Backspace / Delete）

| 测试步骤 | 操作 |
|----------|------|
| 1 | 光标在行中按 Backspace，观察前一字符被删 |
| 2 | 光标在行中按 Delete，观察后一字符被删 |
| 3 | 光标在行首按 Backspace（应合并到上一行） |
| 4 | 光标在空行按 Backspace（应删除空行并合并） |

| 成功边界 | 失败边界 |
|----------|----------|
| Backspace 删除光标前一字符 | 删错字符、无反应 |
| Delete 删除光标后一字符 | 删错字符、无反应 |
| 行首 Backspace 与上一行合并 | 崩溃、只删不合并 |
| 文档首行首列 Backspace 无副作用 | 崩溃、越界 |
| 行尾 Delete 无副作用（不删下一行首字符） | 误删下一行 |

---

### 3.3 换行（Enter）

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在行中按 Enter |
| 2 | 在行尾按 Enter |
| 3 | 在空行按 Enter |

| 成功边界 | 失败边界 |
|----------|----------|
| 当前行从光标处断开，后半部分移到新行 | 内容丢失、顺序错乱 |
| 光标移到新行行首 | 光标留在旧行或错位 |
| 行尾 Enter 产生新空行 | 无新行或产生多余空行 |
| 空行 Enter 产生另一空行 | 崩溃、异常 |

---

### 3.4 插入/覆盖模式

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在 "abc" 的 'b' 前按 Insert 切换为覆盖模式 |
| 2 | 输入 "x"，观察变为 "axc" |
| 3 | 再按 Insert 切回插入模式，光标在 col=2（'x' 后），输入 "y"，观察为 "axyc" |

| 成功边界 | 失败边界 |
|----------|----------|
| 覆盖模式下输入替换当前字符 | 仍是插入 |
| 状态栏显示 OVR / INS 或等价 | 不显示或显示错误 |
| 切换后行为正确变化 | 切换无效 |

---

### 3.5 撤销（Ctrl+Z）

| 测试步骤 | 操作 |
|----------|------|
| 1 | 输入 "hello"，按 5 次 Ctrl+Z |
| 2 | 再按 5 次 Ctrl+Z（无操作后应保持空） |
| 3 | 输入 "x"，再 Ctrl+Z，应回到空 |

| 成功边界 | 失败边界 |
|----------|----------|
| 每次撤销回退一步操作 | 一次撤销多步、或撤销无效 |
| 撤销到空文档后不再改变 | 崩溃、越界 |
| 撤销插入→删除、撤销删除→恢复 | 状态与实际不一致 |
| 撤销换行→两行合并、光标正确 | 合并错误、光标错位 |

---

### 3.6 重做（Ctrl+Y）

| 测试步骤 | 操作 |
|----------|------|
| 1 | 输入 "ab"，Ctrl+Z 撤销，Ctrl+Y 重做 |
| 2 | 再 Ctrl+Y 应无变化 |

| 成功边界 | 失败边界 |
|----------|----------|
| 重做恢复被撤销的操作 | 重做无效、多恢复 |
| 新操作后重做栈清空（可选实现） | 重做产生意外内容 |
| 无重做可执行时 Ctrl+Y 无副作用 | 崩溃 |

---

## 四、光标与导航

### 4.1 方向键

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在多行文本中按 ↑↓←→ |
| 2 | 在第一行按 ↑、最后一行的最后一列按 ↓ |
| 3 | 光标在行首按 ←（预期跳到上一行行尾） |
| 4 | 光标在行尾按 →（预期跳到下一行行首） |
| 5 | 文档首行首列按 ←、文档末行末列按 → |

| 成功边界 | 失败边界 |
|----------|----------|
| ↑ 移到上一行同列（不足则行尾） | 越界、崩溃 |
| ↓ 移到下一行同列（不足则行尾） | 越界、崩溃 |
| ← 在行首时跳到上一行行尾（DOS 标准） | 行首按 ← 无反应、或光标越界 |
| → 在行尾时跳到下一行行首（DOS 标准） | 行尾按 → 无反应、或光标越界 |
| 文档首行首列按 ← 无副作用（无上一行） | 崩溃、负索引 |
| 文档末行末列按 → 无副作用（无下一行） | 崩溃、越界 |

---

### 4.2 Home / End

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在行中按 Home、End |

| 成功边界 | 失败边界 |
|----------|----------|
| Home 移到当前行首列 | 移到错误位置 |
| End 移到当前行尾 | 移到错误位置 |
| 空行时 Home/End 均在 (row, 0) | 列不为 0 |

---

### 4.3 Ctrl+Home / Ctrl+End

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在文档任意位置按 Ctrl+Home、Ctrl+End |

| 成功边界 | 失败边界 |
|----------|----------|
| Ctrl+Home 移到 (0, 0) | 未到文档开头 |
| Ctrl+End 移到最后一行最后一列 | 未到文档结尾 |
| 空文档时两者均在 (0, 0) | 崩溃 |

---

### 4.4 Page Up / Page Down

| 测试步骤 | 操作 |
|----------|------|
| 1 | 在长文档中按 Page Up、Page Down |

| 成功边界 | 失败边界 |
|----------|----------|
| 光标移动约一屏行数（与 edit_rows 一致） | 移动行数错误 |
| 首屏时 Page Up 到 (0, 当前列) | 越界 |
| 末屏时 Page Down 到最后一行 | 越界、超出文档 |

---

### 4.5 Ctrl+← / Ctrl+→（水平滚动视口，仅 WRAP_NONE 模式有效）

| 测试步骤 | 操作 |
|----------|------|
| 1 | 切换到横滚模式（WRAP_NONE），打开超长行文件 |
| 2 | 按 Ctrl+← / Ctrl+→，观察视口水平移动 |
| 3 | 切换到折行模式（WRAP_CHAR），确认 Ctrl+←/→ 无效果 |

| 成功边界 | 失败边界 |
|----------|----------|
| WRAP_NONE 下视口向左/右滚动，可见区间正确更新 | 无反应、崩溃 |
| 滚动后光标保持在可见区域内 | 光标移出屏幕 |
| WRAP_CHAR 下 Ctrl+←/→ 无任何效果 | 折行模式下仍触发滚动或光标乱跳 |
| 到达行首/行尾边界时不继续滚动 | 越界、view_left_col 变为负数 |
